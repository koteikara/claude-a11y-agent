# Web管理画面ユーザーガイド

この文書は、Web管理画面を使ってページの処理、確認、承認を行う人向けの手順書です。基本の説明はリポジトリ直下の [`README.md`](../README.md) にあります。この文書では、Web管理画面だけを使う流れを抜き出して説明します。

## 1. 画面を開く

（ここに画面例：Web管理画面のログイン画面）

1. 管理者から共有された Web管理画面の URL を開きます。
2. 組織のログイン画面が出た場合は、普段使っているアカウントでログインします。
3. ユーザー名とパスワードの入力画面が出た場合は、管理者から受け取った Basic 認証の情報を入力します。
4. 画面上部に `Claude A11y 管理画面` と表示されればログイン完了です。

## 2. ジョブ一覧で対象を探す

（ここに画面例：ジョブ一覧画面）

1. 上部の `ジョブ一覧` を押します。
2. `サイト` にサイト名を入力すると、対象サイトだけを表示できます。
3. `状態` に `queued`、`running`、`done`、`needs_review`、`error` のいずれかを入力すると、その状態だけを表示できます。
4. `更新` を押すと、最新の一覧になります。
5. 一覧は約5秒ごとにも自動で更新されます。

## 3. ジョブを実行する

（ここに画面例：一覧の `実行` ボタン）

1. 実行したい行を探します。
2. 行の右側にある `実行` を押します。
3. 状態が `queued` になり、処理が始まります。
4. 処理中は `running` と表示されます。
5. 完了すると `done`、確認が必要な場合は `needs_review`、失敗した場合は `error` になります。

## 4. 詳細画面を開く

（ここに画面例：ジョブ詳細画面）

1. 確認したい行の `詳細` を押します。
2. 画面上部に `site/page_id` と状態が表示されます。
3. 一覧へ戻る場合は `← 一覧へ` を押します。

## 5. old・ai・gold を見比べる

（ここに画面例：old・ai・gold の3画面比較）

| 表示 | 意味 | 見るポイント |
|---|---|---|
| `old（元ページ）` | 移行前の元ページです。 | 元の内容を確認します。 |
| `ai（AI下書き）` | AI が整えた下書きです。 | リンク文、画像説明、表が分かりやすいか確認します。 |
| `gold（承認済み）` | 人が承認した最終版です。 | 最終版として公開してよいか確認します。 |

1. 3つの枠を左から順に見ます。
2. 文章の意味が変わっていないか確認します。
3. リンクが「こちら」だけになっていないか確認します。
4. 画像の説明が不足していないか確認します。
5. 表に何を示す表か分かる説明があるか確認します。

## 6. 差分を見る

（ここに画面例：差分パネル）

1. `差分` を見ます。
2. `old → ai` では、元ページから AI 下書きで変わったところを確認します。
3. `ai → gold` では、AI 下書きから承認済み版で変わったところを確認します。
4. `追加` は後の版で増えた内容です。
5. `削除` は後の版で無くなった内容です。
6. `変更` は内容または構造が変わった可能性がある行です。

差分は、CMS が自動で付ける飾りをできるだけ除いて、意味のある違いを中心に表示します。

## 7. 要確認を処理する

（ここに画面例：要確認リスト）

1. `要確認` に表示された項目を上から順に読みます。
2. 問題なくそのままでよい場合は `受け入れ` を押します。
3. 修正が必要だと判断した場合は `修正` を押します。
4. 今回は対応しない場合は `スキップ` を押します。
5. 選んだ内容はスプレッドシートの `Review` タブにも保存されます。

## 8. 承認する

（ここに画面例：`承認（gold確定）` ボタン）

1. `old（元ページ）`、`ai（AI下書き）`、差分、要確認を確認します。
2. 最終版として問題なければ `承認（gold確定）` を押します。
3. AI 下書きが gold フォルダへ保存されます。
4. スプレッドシートの `review_status` が `approved` になります。
5. `gold_output_link` に承認済みファイルへのリンクが入ります。

## 9. 指標を見る

（ここに画面例：指標画面）

1. 上部の `指標` を押します。
2. `ジョブ数` で全体の件数を確認します。
3. `未対応の要確認` で、まだ判断されていない要確認の数を確認します。
4. `通過チェック` と `失敗チェック` で確認結果を見ます。
5. 棒グラフで状態ごとの件数を見ます。

## 10. 困ったとき

- `error` が出た場合は、対象の `job_id` とエラー文を管理者へ伝えてください。
- `gold（承認済み）` が空の場合は、まだ承認済みファイルがない可能性があります。
- リンクが開けない場合は、Google Drive の権限が不足している可能性があります。
- ログインできない場合は、URL、表示されたエラー、発生時刻を管理者へ伝えてください。
